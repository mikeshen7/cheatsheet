GitHub Branches

  1. git checkout-b nameOfBranch
    1. Do this in terminal to create a new branch on your computer
    2. checkout moves to branch
    3. -b creates the branch
    4. -d deletes a branch
  2. Make changes in nameOfBranch
  3. ACP to sync, with one change
    1. git add filename
    2. git commit -m "commit notes"
    3. `git push origin nameOfBranch`
    4. This creates a new branch in GitHub
  4. GitHub -> Open Pull Request -> merge it in
    1. This merges branch into main
  5. In terminal git checkout main - moves back to main branch
  6. git pull origin main - pull info from main on GitHub onto local terminal
  7. git checkout nameOfBranch
  8. git merge main - syncs timeline of nameOfBranch to main
  9. git rebase main
    1. Working off another branch (i.e. dev)
    2. rebase main will add any changes in main and include it into dev
  10. git branch -f main C0
    1. moves (by force) main branch to C0 commit
  11. git branch -f main HEAD~3
    1. moves (by force) main branch to 3 parents behind HEAD
  12. git reset
    1. moves a branch to an older commit as if the commit never happened
  13. git revert
    1. Creates a new commit that undoes the changes. Use this if the branch has other branches off of it so you can share those changes
  14. git cherry-pick commit1 commit2
    1. addes changes in commit1 and commit2 to current branch
  15. git rebase -i
    1. -i interactive
